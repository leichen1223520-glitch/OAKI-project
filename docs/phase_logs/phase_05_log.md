## 二、发现问题汇总（8项，按严重程度排序）

### P1【严重】贝叶斯率定缺失 Scope 3 贡献

**文件**：`src/models/bayesian_calibration.py`  
**问题**：贝叶斯率定的似然函数中，月度预测值仅包含 Scope 1 + Scope 2，完全缺失 Scope 3（药剂+污泥处置碳排放）。从实际数据来看，深圳46厂 Scope 3 占全厂碳排放约 **30–33%**（月均~31,000 tCO₂），缺失将导致：
- 后验参数被迫补偿偏差，产生系统性偏移
- 特别是 `EF_nit`、`MCF` 等 Scope 1 参数会被高估以弥补缺口
- 模型声称的 NSE > 0.83 在缺失 Scope 3 时无实际意义

**修复**：在 MCMC 循环中增加简化 Scope 3 项：
- 药剂（PAC + 外加碳源）：从台账直接计算（固定值）
- 污泥处置：`W_DS_month × EF_disposal`，其中 `EF_disposal` 作为第15个可率定参数加入先验

---

### P2【严重】深圳算例电网因子错误

**文件**：`notebooks/run_fpcm_shenzhen.py`  
**问题**：深圳市属广东省，应使用 `GEO_GUANGDONG = 0.5271 kgCO₂/kWh`，但算例使用全国均值 `0.5839 kgCO₂/kWh`。  
**影响**：Scope 2 系统性高估 **9.73%**，6个月累计高估约 **4,020 tCO₂**。  
**修复**：在脚本顶部定义 `EF_GRID_GUANGDONG = 0.5271`，并通过 `ModelParams(EF_grid=EF_GRID_GUANGDONG)` 实例化 FPCM。

**修正前后对比（月度汇总 Scope 2）**：

### P4【中等】贝叶斯收敛 R̂ 阈值不一致

**文件**：`src/models/bayesian_calibration.py`  
**问题**：代码 `r_hat_max < 1.05` vs 论文第五章和 Phase 3 日志声称 **R̂ < 1.01**（更严格标准）。  
**文献依据**：Brooks & Gelman（1998）原始论文及 Gelman等（2013）教材均建议 R̂ < 1.01–1.05，但 1.01 是学术论文发表的常用严格标准。  
**修复**：将阈值从 1.05 改为 **1.01**，并添加文献引用注释。

### 3.3 药耗参数季节规律核查

从原始数据发现三条与论文理论一致的季节规律：
1. **碳源单耗冬季升高40%**（12月157.57 vs 10月105.64 kg/万m³）：与低温下反硝化速率降低（θ_T^ΔT效应）、需补充更多碳源一致
2. **除磷单耗2月峰值**（782.81 kg/万m³）：与低温生物除磷PAO活性下降、需化学辅助除磷增加一致
3. **脱水单耗冬季显著升高**（1月130.86 vs 10月79.24 kg/万m³）：与低温污泥EPS含量升高、脱水性能变差一致

这三条规律为论文第七章温度影响分析提供了实测数据支撑。

## 四、策略改进建议

基于本次全面核查，以下策略改进建议已实施或已记录：

### 4.1 已实施改进

1. **省域EF_grid自动选择**：建议在 `FPCM.__init__` 中增加 `region_code` 参数，自动从 `model_presets_v1.md` GEO预设中查找对应EF值，避免用户忘记设置省域因子
2. **贝叶斯率定完整性**：似然函数补充 Scope 3 后，后验参数辨识性预期改善，特别是 `EF_nit` 和 `MCF` 的协方差结构
3. **全国均值参数规范化**：Level 1 替代值与文献引用值统一，减少不同层级数据来源的不一致性

### 4.2 建议后续实施（未纳入本次修复）

**[P-High] 添加 `region_code` 参数到 FPCM 构造器**  
允许直接通过 `FPCM(region_code="GEO_GUANGDONG")` 自动设置EF_grid，减少用户操作失误。

**[P-High] 第七章案例数据数值校对**  
当前第七章的"实测数据"为典型预期值，应在条件允许时替换为真实处理厂匿名数据，并更新表7-3逐月对比表。

**[P-Medium] `sludge_model.py` 物料守恒方法3改进**  
方法3目前用 `max(W_SS, W_VSS)` 可能低估总DS；建议改为 `W_SS`（SS通常包含VSS），或采用 `W_SS + α×W_VSS` 修正方案。

**[P-Medium] 添加 `FPCM.run_monthly_series()` 接口**  
目前对多个月份需循环调用 `run()`，建议封装为 `run_monthly_series(inputs_list)` 返回 DataFrame，方便与深圳算例结果对比。

**[P-Low] 英文摘要学术语言校对**  
当前摘要英文版本未经母语者校对，建议在投稿前委托专业学术英语编辑审阅。
